# LunaCane 硬件升级采购清单与结构建模方案

更新时间：2026-07-09

## 1. 当前问题判断

当前原型的问题不是单个模块坏，而是工程化程度不足：

- 电池和升压链路抗 WiFi 峰值电流能力不足，容易 brownout、掉线、BMI270 I2C 读失败。
- 模块用胶带缠在拐杖上，线材受力后会松动、虚焊、短路。
- 杜邦线/手焊裸线适合验证，不适合移动场景长期使用。
- 电池、主控、传感器、麦克风、喇叭没有统一固定基准，拆装和维护困难。

升级目标：

- 供电稳定：5V 输入稳定，3.3V 轨就近去耦，WiFi 发包不复位。
- 连接可靠：电源线、信号线、I2C、I2S、按键都用带锁或半锁定连接器。
- 结构可维护：所有模块固定在外壳/内托架上，线材有应力释放。
- 后续可产品化：先做模块化外壳，再根据验证结果考虑定制 PCB。

## 2. 推荐总体硬件架构

建议第一版正式重做仍采用“模块化工程原型”，不要立刻画复杂定制 PCB：

```text
1S 锂电池 / 18650 电池组
  -> 充电保护 / 电源管理模块
  -> 5V 稳压输出
  -> ESP32-S3 主控开发板
       -> 3.3V: BMI270 IMU, I2C 上拉, 低功耗传感器
       -> I2S IN: 数字麦克风
       -> I2S OUT: MAX98357A 功放 + 4Ω/3W 小喇叭
       -> GPIO: 录音按键 / 震动马达 / 指示灯
```

结构上建议分成三个区域：

- 手柄上部：按键、麦克风孔、状态灯，便于交互。
- 手柄/杆身连接处：主控板、电源管理板、线束集中区，便于检修。
- 杆身内部或背夹电池仓：电池和主电源开关，降低头重脚轻。

## 4. 供电方案建议

### 方案 A：最快稳定原型

```text
带保护 1S LiPo / 18650
  -> 5V 2A 充放电一体电源模块
  -> ESP32-S3 5V/VIN
  -> 5V: MAX98357A 功放
  -> 3.3V: BMI270 / I2S 麦克风
```

优点：接线少，适合快速重做。
缺点：要认真挑模块，廉价升压板纹波和瞬态响应可能较差。

关键要求：

- 5V 输出持续能力建议 >=2A。
- 5V 输出端并 470-1000uF 电容。
- ESP32 3V3 和 BMI270 VCC/GND 旁边各放 0.1uF，BMI270 可再加 10-100uF。
- 电源和地线用 22AWG，长度尽量短。

### 方案 B：更工程化

```text
带保护 1S LiPo / 18650
  -> 独立充电管理模块，支持 load sharing 更好
  -> 独立 5V 2A 升压模块
  -> ESP32-S3 / 功放 / 传感器
```

优点：更容易换模块、查问题、测电流。
缺点：模块多，占空间更大。

### 不建议

- 不建议继续使用裸 TP4056 + 廉价小升压模块 + 细杜邦线作为正式方案。
- 不建议把所有模块胶带缠在拐杖上。
- 不建议电池无保护板。
- 不建议电源线和信号线共用细长杜邦线。

## 5. 连接器与线束规范

建议把每个模块定义成可插拔接口：

| 模块 | 推荐连接器 | 引脚 |
|---|---|---|
| 电池 | JST-PH 2P / JST-XH 2P | BAT+, BAT- |
| 主电源输出 | JST-XH 2P / 螺丝端子 | 5V, GND |
| BMI270 | Qwiic/STEMMA QT 或 JST-SH 4P | 3V3, GND, SDA, SCL |
| I2S 麦克风 | JST-SH/PH 5P | 3V3, GND, SCK, WS, SD |
| I2S 功放 | JST-SH/PH 5P + 喇叭 2P | 5V, GND, BCK, WS, DIN |
| 按键 | JST-PH 2P | GPIO, GND |
| 震动马达 | JST-PH 2P | VMOTOR, GND |

线材建议：

- 电池、5V、GND、功放供电：22AWG。
- I2C/I2S/GPIO：26-28AWG。
- I2S 线尽量短，必要时 SCK/WS/DATA 与 GND 绞合，减少干扰。
- 所有焊点必须热缩管保护，线束出口必须有扎带或卡扣做应力释放。

## 6. 外壳与建模方案

### 6.1 第一版结构目标

建议先做“可拆卸电子舱”，不要直接把所有东西塞进拐杖杆内：

- 一个抱箍式/背夹式主控舱，固定在手柄下方 5-12cm 位置。
- 一个可抽拉电池仓，靠近杆身中上部，降低头部重量。
- 手柄处单独开按键孔、麦克风孔、状态灯孔。
- 喇叭朝外或朝前下方开孔，避免被手掌挡住。
- BMI270 固定在靠近拐杖杆体的位置，保持方向稳定，避免软线悬空晃动。

### 6.2 推荐空间布局

```text
手柄顶部/前侧
  [录音按钮] [麦克风孔] [状态灯]

手柄下方电子舱
  [ESP32-S3 主控]
  [BMI270 固定座]
  [I2S 功放]
  [线束整理槽]

杆身侧面/背部电池仓
  [1S 电池]
  [充电/升压模块]
  [主电源开关]
  [USB-C 充电口]

外侧开孔
  [喇叭孔阵列]
  [USB-C 维护/烧录口]
```

### 6.3 建模需要测量的数据

后续正式建模前需要实测：

| 项目 | 需要的数据 |
|---|---|
| 拐杖杆身 | 外径、截面形状、可固定长度、是否可开孔 |
| 手柄 | 长宽高、曲面位置、可安装按键区域 |
| ESP32-S3 板 | 长宽高、USB 口位置、安装孔距 |
| 电池 | 长宽厚或 18650 数量/排列方式 |
| 电源模块 | 长宽高、USB-C 口位置、安装孔 |
| 喇叭 | 直径、厚度、孔距、出声方向 |
| 麦克风 | 板尺寸、麦克风孔位置 |
| BMI270 | 板尺寸、安装孔距、坐标方向 |

### 6.4 我可以继续帮你做的建模交付

只要你们给出上述尺寸，我可以继续做：

- 参数化 OpenSCAD 外壳模型：电子舱、电池仓、抱箍、螺丝柱、出线槽。
- 3D 打印 STL 导出流程说明。
- 模块布局图和线束连接图。
- 后续如果要更正式，可以再做 KiCad 底板原理图/PCB，把 ESP32、BMI270、麦克风、功放、电源接口固定到一块底板上。

建议第一版外壳采用：

- 材料：PETG 优先，PLA 可用于展示但耐热和抗冲击较弱。
- 壁厚：2.0-2.5mm。
- 螺丝：M2 或 M2.5。
- 结构：上下壳分体，4 颗螺丝固定。
- 与拐杖连接：双抱箍 + 橡胶垫，避免直接打孔削弱拐杖强度。

## 7. 第一轮采购优先级

如果预算有限，优先买这些：

1. 可靠电池 + 5V 2A 电源管理模块。
2. 22AWG/26AWG 硅胶线、JST 连接器、热缩管。
3. 470-1000uF 电容、0.1uF 电容、4.7kΩ 电阻。
4. 螺丝铜柱、3D 打印材料、扎带/线夹。
5. 备用 ESP32-S3、BMI270、I2S 麦克风、MAX98357A。

第一轮重做的验收标准：

- 电池供电下连续 WiFi 上传 30 分钟不复位。
- BMI270 50Hz 采集无连续 I2C 失败。
- 按键录音和喇叭播放时系统不掉电。
- 拐杖正常挥动、轻敲、移动后接线不松动。
- 电池、主控、传感器可以拆开维护，不需要撕胶带。
